# bioFlow — Design Context

## Users

Computational biologists and bioinformaticians running single-cell genomics,
scRNA-seq, and related analyses. They spend multi-hour sessions alternating
between writing Python/R code, watching AI agents work, and reading dense
tracebacks, tables, and plots. The interface is a cockpit, not a consumer
app — density, speed, and long-session comfort outrank friendliness.

Context of use: desktop browser (laptop or external monitor), usually
evenings or focused deep-work blocks. Often paired with a terminal and the
lab notebook side-by-side.

## Brand Personality

Three words: **precise, deliberate, scientific.**

Not: friendly, playful, consumer-app-cheerful, corporate-tech. Think
lab notebook, research instrument, typographic academic journal — *not*
"AI assistant for everyone".

Emotional goal: confidence that the tool respects the user's expertise and
time. A scientist-grade instrument that does not patronize.

## Aesthetic Direction

- **Dark theme primary.** Hours-long staring at code, plots, and tracebacks —
  glare is the enemy. A warm-slate dark (not GitHub-blue dark).
- **Editorial typographic voice.** One distinctive display face for emphasis,
  one humanist sans for the UI, purpose-built mono for code. Hierarchy does
  the work that chrome usually does.
- **Earned accents.** Color carries meaning: running, succeeded, failed,
  interactive. Anything else stays neutral. Small area, high conviction.
- **Information density is fine.** A chat + cells + variables + files layout
  is crowded on purpose — users want to see state, not click to reveal it.
- **No AI-slop tells.** No gradient text. No left-stripe accent borders on
  cards. No glassmorphism. No generic rounded-card grids.

Anti-references: stock "AI assistant" dashboards, GitHub's default palette,
Linear's violet gradient, any "friendly data" cutesy style.

## Design Principles

1. **Data legibility first.** If a typographic or color choice makes code,
   numbers, or tracebacks harder to read, it's wrong — regardless of how
   it looks.
2. **Earn every color.** An accent means *something*. If a button is
   accent-blue for no reason, recolor it neutral.
3. **Typography carries the hierarchy.** Size and weight do 80% of the work.
   Borders, backgrounds, and icons are the remaining 20%.
4. **Neutrals are tinted, never pure.** Pure `#000`/`#fff` and the GitHub
   blue-gray are both off-limits. Neutrals inherit a subtle warm-green
   tint from the brand hue.
5. **Intentional, not generic.** If a designer could look at the UI and
   immediately say "AI made this", redesign the detail.
